Learning Git and Github

Shell commands

  1. pwd know the path of the folder
  2. ls know the files under the folder
  3. cd changing directory of the files 
    cd subfolder
    The file will be changed to the subfolder under the same path
  4. nano edit a file: delete, add or change content
    1. ctrl+0 save changes
    2. ctrl+x exit editor
  5. Editing a file
    1. echo—create or edit a file
    2. Create a new file todo.txt
       echo "Review for duplicate records" > todo.txt
  1. Add content to existing file todo.tx
   echo "Review for duplicate records" >> todo.txt

Git

Work with repository

  1. Create a git repo
#Start sth new
git init repo_name
#Already in a project? Create one based on this
git init
#Move to this directory
git cd new_repo
#Create a file for this new repo
git add file.txt

Basic: From Create to Push

  1. Check Git version
git ---version
  1. Modify a file
nano yourfile.Md
  1. Add file to stage
git add yourfile.md
git add file1.md file2.py #Stage multiple files

Add all the file to the stage

git add .

.= all the files in the current location Add the same type of file (all the py file) to stage

git add *.py
  1. Make a commit
git commit -m "update in the file"
  1. Check the status of the file
git status
  1. Initialize git repo
git Init

If this is the first time and haven’t configure Git

git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"
  1. After changing the file to stage (which means the local repo), create a repo on your github, copy the url of your repo and add the remote url
git remote add origin https://github.com//theurlofrepo
  1. Push to github main/master branch
git push -u origin master

Collaborate on Github

  1. Clone project
#Clone from github
git clone path_project
#Clone from local to local
git clone home/gitfile/old_repo new_repo_name
  1. Create a remote
git remote
# Get details
git remote -v
#Name the remote
git remote add repo_name url
  1. Fetch from remote Important to save locally before pull
#Fetch from main
git fetch origin main
#Fetch from other branch
git fetch origin report
#Synchronizing content
git merge origin main
# Pull command is the combination of fetch and merge
git pull origin main
  1. Push to a remote

Comparison Between Different Commit

  1. Compare the unstaged file with last commit diff
git diff filename.md

Compare staged file with the last commit

git diff -r HEAD yourfile.md

Compare all staged file with the last commit

git diff -r HEAD
  1. History of commit/ How to find the commit that causes the issue
    1. Find the particular commit
git log #All the commit history
git log -3 # 3 most recent history
git log -3 readme.md
git log --since='Apr 24 2024' --until='Oct 10 2024'

Press space to show more recent commits Press q to quit the log and return to the terminal This will return the hash number for each commit, the author and the commit

ii. Only need the first 6-8 characters of the hash

git show 45268e
  1. Show the diff between two commit HEAD~1 is the second most recent commit
git diff -r HEAD HEAD~2

HEAD Is the latest commit HEAD Is the two commits before the latest commit

View the changes of a file by line

git annotate file.md

  1. Undo Changes before commit
    1. Unstaging files
git reset HEAD file.md #git reset HEAD
nano file.md 
git add file.md
git commit -m""

Default for reset is set to --mixed, which means unstaging the file You can use soft to amend commit messages

 git reset --soft HEAD~1 #the HEAD is now pointed to HEAD~1, the latest commit is now staged for any amend
  1. Undo changes to unstaged files 
    git checkout -- file.csv
git checkout . #Undo all the changes for all files unstaged
git checkout dasd545 file.csv #replace the -- with the commit hash to restore to a specific version of commit
git checkout HEAD~2 file.csv #same as above

checkout means switching to a different version Defaults to the last commit This means losing all changes made to the unstaged file forever

  1. Cleaning a repository See what files are not being tracked:
git clean -n #

Delete those files:

git clean -f

git clean -f ==-cannot be undone!-==

Configure Git

git config –list Git has three levels of settings:

  1. --local: settings for one specific project

  2. --global: settings for all of our projects

  3. --system: settings for every users on this computer

  4. Change Global setting

git config --global setting name value
# i.e: git config --global user.address sher@gmail.com

change name:

git config --global user.name 'Sher Liu'
#For names use ' ' or git will only use the name before the space
  1. Set alias for a command We can shorten the command by giving them an alia ie: Shorten commit command
git config --global alias.ci 'commit --m' #ci is the alias you choose
#In the future you can use git ci instead of git commit --m

Track the alias:

git config –global –list

  1. Ignore files You can tell it to stop paying attention to files you don’t care about by creating a file in the root directory of your repository called .gitignore

  2. Branches Every box is a commit Here we have 2 branches, Analysis and Report After completed modification in each branch we can merge them to the main

    1. Check branch
git branch

The branch with a * is the branch we are on

  1. Create a new branch
git checkout -b new_branch_name
  1. Diff between branch
git diff main new_branch
  1. Switch to a branch
git checkout branch_name
  1. Merge Branch
git merge source destination
  1. Avoid Conflict If two branches are editing the same file, which usually we should avoid. We might experience conflicts when merging the branches to the main.

Example of outcome in conflict:

<<<<<<< HEAD
// Your changes (from the current branch)
=======
// Incoming changes (from the branch being merged)
>>>>>>> branch-name

<<<<<<< HEADMeaning: Start of the conflicting changes from the current branch (the branch you’re on). =======Meaning: Separator between your changes and the incoming changes. >>>>>>>branch-name Meaning: End of the conflicting changes from the incoming branch (the branch you’re merging into your current branch).
